## Overview

Coffetto uses modern CSS with custom properties (CSS variables), mobile-first responsive design, and a sophisticated color system. The stylesheet is organized with clear sections and follows a structured cascade from base styles to component-specific rules.

### Architecture Highlights
- CSS Custom Properties in `:root`
- Mobile-first responsive approach
- Google Fonts integration (Poppins & Pacifico)
- Flexbox and Grid layouts
- Smooth transitions and hover effects
- ScrollReveal animation support

---

## CSS Custom Properties

All design tokens are defined in the `:root` selector for easy theming and consistency:

```css styles.css
:root {
  --header-height: 3.5rem;
  --border-section: 56px 56px 0 0;

  /*========== Colors ==========*/
  --first-color: hsl(12, 32%, 22%);
  --first-color-alt: hsl(12, 42%, 12%);
  --first-color-light: hsl(12, 24%, 32%);
  --second-color: hsl(39, 62%, 65%);
  --title-color: hsl(12, 12%, 16%);
  --text-color: hsl(12, 12%, 32%);
  --text-color-light: hsl(12, 8%, 48%);
  --white-color: #fff;

  /*========== Font and typography ==========*/
  --body-font: "Poppins", sans-serif;
  --second-font: "Pacifico", cursive;
  --biggest-font-size: 2.75rem;
  --big-font-size: 1.75rem;
  --h1-font-size: 1.5rem;
  --h2-font-size: 1.25rem;
  --h3-font-size: 1rem;
  --normal-font-size: .938rem;
  --small-font-size: .813rem;
  --smaller-font-size: .75rem;

  /*========== Font weight ==========*/
  --font-regular: 400;
  --font-medium: 500;
  --font-semi-bold: 600;

  /*========== z index ==========*/
  --z-tooltip: 10;
  --z-fixed: 100;
}
```

<Accordion title="Understanding the Custom Properties">
  - **Layout variables**: `--header-height` and `--border-section` for consistent spacing
  - **HSL color format**: Enables easy manipulation of hue, saturation, and lightness
  - **Font scaling**: Mobile-first sizes that scale up on larger screens
  - **Z-index system**: Predefined layers prevent z-index conflicts
</Accordion>

## Color System

The color palette is built around two primary colors with variations:

<Tabs>
  <Tab title="Primary Colors">
    ```css
    --first-color: hsl(12, 32%, 22%);        /* Dark brown - main brand color */
    --first-color-alt: hsl(12, 42%, 12%);    /* Darker brown - for gradients */
    --first-color-light: hsl(12, 24%, 32%);  /* Lighter brown - accents */
    --second-color: hsl(39, 62%, 65%);       /* Golden yellow - CTAs & highlights */
    ```
    
    The brown tones create a coffee-themed aesthetic, while the golden yellow adds warmth and draws attention to interactive elements.
  </Tab>
  <Tab title="Text Colors">
    ```css
    --title-color: hsl(12, 12%, 16%);        /* Almost black for headings */
    --text-color: hsl(12, 12%, 32%);         /* Dark gray for body text */
    --text-color-light: hsl(12, 8%, 48%);    /* Medium gray for secondary text */
    --white-color: #fff;                     /* Pure white */
    ```
    
    Text colors maintain the warm brown undertone while ensuring readability.
  </Tab>
</Tabs>

<Note>
  Using HSL (Hue, Saturation, Lightness) instead of hex or RGB makes it easier to create color variations by adjusting individual values.
</Note>

## Typography System

Coffetto uses two Google Fonts for visual hierarchy:

```css styles.css
@import url("https://fonts.googleapis.com/css2?family=Pacifico&family=Poppins:wght@400;500;600&display=swap");
```

### Font Families

| Variable | Font | Usage |
|----------|------|-------|
| `--body-font` | Poppins | Body text, buttons, navigation |
| `--second-font` | Pacifico | Headings, logo, decorative text |

### Font Size Scale (Mobile)

```css
--biggest-font-size: 2.75rem;   /* 44px - Hero titles */
--big-font-size: 1.75rem;        /* 28px - Section titles */
--h1-font-size: 1.5rem;          /* 24px - H1 headings */
--h2-font-size: 1.25rem;         /* 20px - H2 headings */
--h3-font-size: 1rem;            /* 16px - H3 headings */
--normal-font-size: .938rem;     /* 15px - Body text */
--small-font-size: .813rem;      /* 13px - Small text */
--smaller-font-size: .75rem;     /* 12px - Tiny text */
```

### Responsive Typography

Font sizes scale up on desktop:

```css styles.css
@media screen and (min-width: 1024px) {
  :root {
    --biggest-font-size: 4.5rem;     /* Increases from 2.75rem */
    --big-font-size: 3rem;            /* Increases from 1.75rem */
    --h1-font-size: 2.25rem;
    --h2-font-size: 1.5rem;
    --h3-font-size: 1.25rem;
    --normal-font-size: 1rem;
    --small-font-size: .875rem;
    --smaller-font-size: .813rem;
  }
}
```

## Base Styles

Foundation styles applied globally:

```css styles.css
* {
  box-sizing: border-box;
  padding: 0;
  margin: 0;
}

html {
  scroll-behavior: smooth;
}

body {
  font-family: var(--body-font);
  font-size: var(--normal-font-size);
  background-color: var(--white-color);
  color: var(--text-color);
}

ul {
  list-style: none;
}

a {
  text-decoration: none;
}

img {
  max-width: 100%;
  height: auto;
}
```

<Note>
  `scroll-behavior: smooth` enables smooth scrolling for anchor links without JavaScript. The universal box-sizing reset ensures consistent sizing across all elements.
</Note>

## Reusable Utility Classes

Common layout patterns extracted into utilities:

```css styles.css
.container {
  max-width: 1120px;
  margin-inline: 1.5rem;
}

.grid {
  display: grid;
  gap: 1.5rem;
}

.section {
  padding-block: 4.5rem 3.5rem;
}

.section__title {
  color: var(--title-color);
  font-size: var(--big-font-size);
  font-family: var(--second-font);
  font-weight: var(--font-regular);
  line-height: 120%;
  text-align: center;
  margin-bottom: 1.5rem;
}
```

<Accordion title="Utility Class Breakdown">
  - **`.container`**: Centers content with max-width and horizontal margins
  - **`.grid`**: Enables CSS Grid with consistent gap
  - **`.section`**: Vertical padding for content sections
  - **`.section__title`**: Styled heading used across multiple sections
</Accordion>

## Header & Navigation

### Fixed Header

```css styles.css
.header{
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  background-color: var(--first-color);
  z-index: var(--z-fixed);
  transition: background .4s, box-sadow .4s;

.header__border{
  width: 100%;
  height: 2px;

                    var(--text-color-light) 50%,
                    transparent 50%);
  background-size: 24px;
  position: absolute;
  bottom: 0;
  left: 0;
  transition: opacity .3s;
}
```

The header border creates a dashed line effect using a repeating linear gradient.

### Navigation Links

```css styles.css
.nav__link{
  color: var(--white-color);
  font-family: var(--second-font);
  font-size: 10px;
  display: flex;
  flex-direction: column;
  align-items: center;
  row-gap: .25rem;

  background: linear-gradient(180deg,
              hsl(39, 62%, 75%) 0%,
              hsl(39, 62%, 45%) 100%);
  color: transparent;
  -webkit-background-clip: text;

.nav__link:hover{

  color: var(--second-color);

```

<Note>
  The icons use `background-clip: text` to create a gradient effect. The `color: transparent` makes the text transparent so the background gradient shows through.
</Note>

## Button Component

A reusable button with gradient, shadow, and icon:

```css styles.css
.button{
  display: inline-flex;
  align-items: center;
  column-gap: .5rem;
  background-color: var(--second-color);
  padding: 1rem 1.5rem;
  border-radius: 4rem;
  color: var(--title-color);
  font-weight: var(--font-medium);
  box-shadow: 0 8px 32px hsla(39, 62%, 65%, .25);
  transition: box-shadow .3s;
}

.button i{
  font-size: 1.5rem;
  transition: transform .3s;
}

.button:hover{
  box-shadow: 0 8px 48px hsla(39, 62%, 65%, .3);
}

.button:hover i{
  transform: translateX(.25rem);
}
```

<Accordion title="Button Hover Effects">
  1. **Shadow expansion**: Box-shadow blur increases from 32px to 48px
  2. **Icon animation**: Arrow icon slides 0.25rem to the right
  3. **Smooth transitions**: Both effects use 0.3s timing
</Accordion>

```css styles.css
.scrollup{
  position: fixed;
  right: 1rem;

              var(--first-color) 0%,
```

The button starts at `bottom: -50%` (hidden), then animates to `bottom: 7rem` when `.show-scroll` is added.

## Responsive Design Strategy

Coffetto uses a mobile-first approach with three main breakpoints:

<Tabs>
  <Tab title="Small Devices (<300px)">
    ```css
    @media screen and (max-width: 300px) {
      .container{
        margin-inline: 1rem;

        width: 220px;
    ```

## Animation & Transitions

Smooth transitions are used throughout:

```css
/* Header transitions */
.header{
  transition: background .4s, box-sadow .4s;
}

/* Navigation link hover */
.nav__link{
  transition: color .3s;
}

/* Button hover effects */
.button{
  transition: box-shadow .3s;
}

.button i{
  transition: transform .3s;
}

/* Image hover */
.products__img{
  transition: transform .4s;
}

/* Scroll button reveal */
.scrollup{
  transition: bottom .5s, transform .3s;
}
```

<Accordion title="Transition Timing">
  - **0.3s**: Quick interactions (hover, active states)
  - **0.4s**: Medium transitions (background changes, image effects)
  - **0.5s**: Slower animations (scroll button reveal)
</Accordion>

## Key Takeaways

1. **CSS Custom Properties**: All design tokens in `:root` enable easy theming
2. **Mobile-First**: Base styles target mobile, media queries progressively enhance
3. **HSL Colors**: Easier to create variations than hex/RGB
4. **Utility Classes**: Reusable patterns reduce CSS bloat
5. **Grid & Flexbox**: Modern layout techniques throughout
6. **Smooth Transitions**: Polished feel with consistent timing
7. **Responsive Typography**: Font sizes scale with viewport

For implementation details, see [HTML Structure](/components/html-structure) and [JavaScript Features](/components/javascript-features).
