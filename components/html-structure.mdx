## Overview

The Coffetto website uses semantic HTML5 elements with a well-organized structure. The HTML is written in Spanish ("Sitio web de café receptivo - YoelFH") and features a single-page layout with smooth scrolling navigation.

### Key Features
- Semantic HTML5 elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`)
- Fixed header with bottom navigation on mobile
- Single-page architecture with section-based navigation
- Remixicon integration for icons
- ScrollReveal library for animations

---

## Document Head

The document head includes essential meta tags and external resources:

```html
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!--=============== FAVICON ===============-->
    <link rel="shortcut icon" href="assets/img/favicon.png" type="image/x-icon">

    <!--=============== REMIXICONS ===============-->
    <link href="https://cdn.jsdelivr.net/npm/remixicon@2.5.0/fonts/remixicon.css" rel="stylesheet">

    <!--=============== CSS ===============-->
    <link rel="stylesheet" href="assets/css/styles.css">

    <title>Sitio web de café receptivo - YoelFH</title>
</head>
```

<Note>
  The viewport meta tag ensures proper responsive behavior across devices, while Remixicon provides a comprehensive icon library for the UI.
</Note>

### Navigation Structure

<Accordion title="Header Components Breakdown">
  - **`header__border`**: Decorative 2px gradient border at the bottom of the header
  - **`nav__logo`**: Logo with image and text combination
  - **`nav__menu`**: Navigation menu (positioned at bottom on mobile)
  - **`nav__list`**: Unordered list of navigation items
  - **`nav__link`**: Each link contains an icon and text span
  - **`active-link`**: Class added to highlight the current section
</Accordion>

<Note>
  The navigation uses anchor links (`#home`, `#about`, etc.) that scroll to corresponding sections. JavaScript handles the active state highlighting.
</Note>

## Class Naming Convention

Coffetto follows the **BEM-like** naming pattern:

- **Block**: `.home`, `.products`, `.footer`
- **Element**: `.home__container`, `.nav__link`, `.footer__social-link`
- **Modifier**: `.active-link`, `.steps__card-move`, `.scroll-header`

### Common Utility Classes

| Class | Purpose |
|-------|--------|
| `.container` | Max-width wrapper (1120px) with horizontal margin |
| `.grid` | Display grid with 1.5rem gap |
| `.section` | Vertical padding (4.5rem top, 3.5rem bottom) |
| `.section__title` | Large decorative title using Pacifico font |
| `.button` | Primary CTA button with icon |

## Script Loading

Scripts are loaded at the end of the body for optimal performance:

```html
<!--=============== SCROLLREVEAL ===============-->
<script src="assets/js/scrollreveal.min.js"></script>

<!--=============== MAIN JS ===============-->
<script src="assets/js/main.js"></script>
```

<Note>
  ScrollReveal library must load before `main.js` since the main script depends on the `ScrollReveal` object being available.
</Note>

## Semantic HTML Summary

The structure uses appropriate semantic elements:

- `<header>` - Site header with navigation
- `<nav>` - Navigation wrapper
- `<main>` - Main content container
- `<section>` - Distinct content sections (home, products, about, steps, testimonial)
- `<article>` - Self-contained product cards
- `<footer>` - Site footer

This semantic approach improves accessibility, SEO, and code maintainability.

<Note>
  All sections have unique `id` attributes matching the navigation anchor links, enabling smooth scroll navigation and active link highlighting.
</Note>
